# 完整跑通示例：S4 论文的 Introduction/GAP（真实论文验证版）

这版示例直接用 S4 论文（Efficiently Modeling Long Sequences with Structured State
Spaces）的真实 Introduction 内容跑通，并演示了 `Y`、`old_structure`、`new_form` 三个
可选字段省略后的效果——这篇论文的"核心问题"和"研究目标"本质上是同一件事，"本文工作"
也是直接从机制跳到效果，没有"把旧结构重新表述成新形式"这一步，所以三个可选字段都不填。

## 第 1 步：生成骨架

```bash
python scripts/init_skeleton.py --out intro.json
```

## 第 2 步：读 Introduction 后整理出的 values.json（省略了可选字段）

```json
{
  "stage1": {
    "X": "序列建模",
    "goal_verb": "高效捕捉",
    "Z": "长距离依赖",
    "A": "RNN、CNN、Transformer 以及状态空间模型（SSM）"
  },
  "stage2": {
    "B": "超长序列建模场景",
    "limitations": [
      "Transformer：计算复杂度过高",
      "RNN：难以捕捉远距离依赖",
      "CNN：感受野受限",
      "SSM：虽具理论优势但实际计算难以高效实现"
    ],
    "core_problem": "高效长序列建模",
    "req1": "长程依赖建模能力",
    "req2": "计算效率"
  },
  "stage3": {
    "M": "S4（结构化状态空间模型）",
    "mechanism": "对SSM进行结构化参数化设计，使状态矩阵可稳定对角化，并将序列计算转化为高效核形式"
  }
}
```

注意 stage1 里没有 `Y`，stage3 里没有 `old_structure`/`new_form`——values.json 里直接
不写这几个 key，脚本会自动切换到精简版句式，不会因为字段必填而硬凑一句重复的话。

```bash
python scripts/fill_intro.py --skeleton intro.json --values values.json --out intro_filled.json
```

## 第 3 步：渲染段落 + GAP 表 + 判定自检

```bash
python scripts/render_intro.py --in intro_filled.json
```

输出三段：

> ①在序列建模中，研究目标是理解或构建能够高效捕捉长距离依赖的理论或方法。该领域已有多种研究路径或方法，包括RNN、CNN、Transformer 以及状态空间模型（SSM），它们分别从不同角度尝试解决该问题。
>
> ②然而，在超长序列建模场景下，这些方法仍然存在不足，主要表现为：Transformer：计算复杂度过高、RNN：难以捕捉远距离依赖、CNN：感受野受限、SSM：虽具理论优势但实际计算难以高效实现。因此，在高效长序列建模上仍存在尚未解决或解释不足的研究缺口（GAP），即现有理论或方法无法同时满足长程依赖建模能力与计算效率。
>
> ③针对该 GAP，本文提出S4（结构化状态空间模型），通过对SSM进行结构化参数化设计，使状态矩阵可稳定对角化，并将序列计算转化为高效核形式，从而实现对长距离依赖的改进或提升。

## 什么时候该填可选字段，什么时候该省略

- **Y（核心问题）**：如果"核心问题"和"研究目标"实质上是同一句话的两种说法（很多论文都是
  这样），就不填 Y，让脚本只渲染一句话，不要硬凑出两句重复的内容。只有当核心问题和研究
  目标确实有明显区别时（比如核心问题是"现象为什么发生"，研究目标是"如何利用这个现象"）
  才需要把 Y 单独写出来。
- **old_structure / new_form（重新表述/转化）**：如果本文工作就是"通过机制直接带来效果"，
  没有一个明确的"把旧问题结构转化成新形式"的步骤，就两个都不填。只有当论文确实有这层
  重新建模/重新表述的动作时（比如把回归问题转化成分类问题、把递归形式转化成卷积形式）才
  把这两个字段填上。

## 小提示
渲染结果是机械拼接的，中英文混排处偶尔会缺一个空格。这是预期的——脚本只保证结构、变量和
故事线闭环都对，最后过一遍微调断句是人的工作。
