# 中文学术规则

仅在可编辑正文以中文为主时加载。语义合同、术语合同、全稿模式合同、学术
白名单、虚假对立规则和 `SKILL.md` 的统一仲裁顺序始终优先。

## 目录

1. 启动标准

## 1. 启动标准

规则有两个启动层级：单个表达已经空转、缺乏来源或改变语义时，按局部缺陷处理；
单个表达可以成立但在段落、章节或全稿中机械占优时，按分布缺陷处理。一个词或
句式经常出现在模型文本中，不代表它在当前论文中有问题，也不能证明作者身份。

启动规则前依次判断：

- 这段话承载了什么命题或逻辑关系？
- 命题能否回指用户提供的正文、数据或来源？
- 这种表达是否符合当前章节和学科惯例？
- 修改后能否保持否定、模态、因果强度、比较对象和范围？
- 问题来自这个表达本身，还是来自它在完整文本中的重复位置和功能？

无法确定时保留或标记，不为制造变化而变化。

## 2. 中文模式

### Z1. 模板化开场

候选包括“随着……的不断发展”“在当今……背景下”“近年来……日益受到广泛
关注”。没有具体事实或引文时，直接进入研究对象；有明确时间、趋势、文献或
问题界定时保留。

### Z2. 空泛拔高与句尾升华

候选包括“具有重要意义”“标志着新阶段”“开启新篇章”“提供有力支撑”“奠定
坚实基础”“彰显了”“反映了”。删去后命题不变，说明它只是评价。删除评价或
直陈来源中已有的结果，不得补充数字、影响、应用或限制。若多个句尾或段尾都
执行同一升华动作，按 `global-pattern-contract.md` 检查位置规律和功能冗余。

基金和政策文本中的规定性表达按白名单处理，不得机械清除。

### Z3. 无来源的修辞性对立

候选包括“不是 X，而是 Y”“不仅仅是 X，更是 Y”“问题不在于 X，而在于 Y”
“表面上是 X，本质上是 Y”“与其说 X，不如说 Y”。

必须执行 [虚假对立规则](contrast-logic.md)。核心判据是 X 的来源和论证功能，
不是 Y 是否具体，也不是句式出现次数。真实反驳、消融归因、负结果、定义、范围
界定和审稿回复予以保护；具体但缺乏证据的对立只标记，不自动改。加法结构中的
X、Y 均有内容时，不得删掉任何一项。

### Z4. 英文直译腔

候选包括机械的“作为一个 X”开场、过量主语代词、“这关于……”“这不仅仅关于
……”以及不符合中文信息顺序的长前置修饰。按中文话题和状语顺序重写，但保持
主客体、否定、范围和因果关系。

英文术语、引文和作者有意采用的跨语言表达不在此列。

### Z5. 装饰性强调与证据失配的确定性

候选包括没有证据支撑的“极大地”“深刻地”“前所未有地”“显著地”“全面地”，
以及“一定”“必然”“毫无疑问”“无疑”等确定性表达。功能性幅度词、有统计
依据的“显著”、定义、定理和已证明不变量中的必然性保留。

确定性必须与命题账本的证据强度一致。来源只支持探索性关联时，恢复其原有模态
和关系；来源缺失时保留并标记，不能自行猜测一个较弱结论。

### Z6. 商业和宣传黑话

候选包括“赋能”“抓手”“闭环”“底层逻辑”“一站式”“全方位”“开启新赛道”。
仅在它们是空泛隐喻时改为来源已有的动作、机制或对象；政策原文、研究对象原话和
领域内定义用法受保护。

### Z7. 机械分点、三项包装与穷举式周全

连续论证被“首先、其次、最后”或编号强行切开，且各点并非真正并列时，恢复为
连贯段落。方法步骤、贡献列表、评价维度和真实枚举保留，不改变项目数量。

三个形容词或三个抽象优点连续出现，只在它们可互换、缺少来源或反复充当完整性
装饰时处理。对“覆盖所有情况”的平铺清单，先找来源中已有的维度、因果或层级；
只有层级确实存在时才据此组织。不得为了优雅而发明分类、合并条件或删去例外。

### Z8. 标点和插入语

检查装饰性破折号、省略号、半角标点混入中文正文和过长插入语。能在不损失关系
时可改为逗号、括号、冒号或独立句。公式、范围、引文和作者有意使用的标点保留。

不设置统一“接近零”目标，也不以单个破折号判定问题。
破折号频率本身不是稳定的模型作者标记。多段重复时只把位置分布作为候选证据，
仍需确认每处是否承载必要的限定、转折或插入关系。

### Z9. 万能结尾与段落峰值饱和

候选包括“未来可期”“任重而道远”“机遇与挑战并存”“需要多方共同努力”“为
后续研究奠定基础”。没有具体内容时删除；来源已提供明确结论、边界或开放问题时
以其收束。不得自行补一个合理的研究限制或未来方向。

若每段都以“金句”、抽象判断或重要性宣告收束，检查这些句子是否推进、综合或
限制了论证。保留有证据且承担综合功能的句子；删除只重复重要性的段尾，不按配额
保留金句。

### Z10. 四字词堆叠和局部称呼轮换

- 宣传性四字词连续堆叠且不增加信息时，改为直接陈述。
- 普通指称在局部无理由更换多个称呼并造成阅读障碍时，恢复稳定指称。

保留学科固定搭配、引文和对不同群体的真实区分。论文全局的科学术语一致性、
新命名、全称与缩写、跨语言映射及有意区分，统一执行
`terminology-contract.md`；不得合并真实不同的模型、模块、目标函数、变量或指标。

### Z11. 助手语气和过程残留

“希望对您有帮助”“以下是修改后的版本”“按照要求本文避免……”等属于过程层，
不进入成品。若同句包含正文命题，只移除过程片段并重建衔接。

引文、问卷项目和对 AI 文风的学术讨论适用“提及而非使用”豁免。

### Z12. 格式、来源和占位符残留

处理与目标格式不符的 Markdown 标记、工具引用标记和链接跟踪参数。占位符只标记
待核对，不能猜测填充。可疑引文在正文外提示核验，不得自行改作者、年份或 DOI。

### Z13. 句首路标词的机械复用

候选包括连续或跨段反复以“此外”“然而”“值得注意的是”“具体而言”等开头。
连接词只有在明确加法、转折、强调或细化关系时才保留。关系已由结构和内容清楚
表达时，可合并句子、调整信息顺序或删除冗余路标，但不能删除关系本身。

不按词表封禁连接词。使用 `global-pattern-contract.md` 同时检查次数、分布、位置和
功能；一个合法句首或多次承担不同逻辑功能的句首不构成缺陷。

### Z14. 机械结构、句子节奏和段落模板

检查连续同构句、重复开头、每段都执行相同“背景—三点—总结”模板等现象。只有
机械结构影响论证时才调整。方法步骤、平行实验结果、定义段和短段落不需要人为
打乱。

按章节功能查看有序句长序列，而非只看平均句长。全长句、全短句或机械长短交替
只有在违背命题边界、降低可读性时才调整。句子拆分和合并必须服从命题及关系，
不设置每段必须出现长短句、首句必须立论或每句必须有显式连接词的配方。

## 3. 生成期约束

1. 从用户提供的研究对象、问题、方法或结果进入，不写空泛时代背景。
2. 不凭空建立一个 X 再否定，以制造“不是 X，而是 Y”的深刻感。
3. 合法连接词、hedging、必要被动、`进行+V` 和术语按功能使用。
4. 句法随内容变化，不按字数配方制造节奏。
5. 只在内容真正可枚举时使用列表。
6. 以已有结论、边界或问题结束；内容说完即可结束，不补仪式性总结。
7. 假设必须明示，缺失事实必须询问或保留为用户认可的占位符。
8. 在继续论证前核对引用、结果和事实前提的来源状态；草稿中的陈述不能自证。

## 4. 最终语言检查

- 没有新增事实、归因、比较、机制、意义、应用、限制或未来工作。
- 没有在缺乏证据时删除具体否定或对立命题。
- 合法 hedging、被动、连接关系、领域术语和中文学术形态均被保留。
- 空泛拔高、宣传黑话和仪式性表达已在不降语域的前提下处理。
- 局部与全稿模式合同均已执行；不存在未经功能复核的连接词、三项并列、破折号、
  段尾升华、确定性或机械模板清零目标。
- 全稿术语合同已通过，普通指称稳定，标点符合目标格式。
- 成品不含过程说明、助手自述、工具残留和猜测填充的占位符。
