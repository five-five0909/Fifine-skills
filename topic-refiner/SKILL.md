# Topic Refiner Skill

## 一句话描述
把宽泛的研究方向或已有成果，转化为一个有问号、有谜题、可回答的研究问题。

## 触发条件
以下任意一种情况触发本技能：

**选题类**
- "帮我选题" / "题目太大了" / "不知道怎么切入"
- "我想研究XXX，但没有方向" / "论文题目怎么聚焦"
- "选题太宽" / "题目怎么缩小"

**叙事类**
- "帮我优化引言" / "论文怎么讲故事"
- "我有实验结果，但不知道怎么写" / "审稿人说没有contribution"
- "我的论文像方法介绍，不像回答问题"

**诊断类**
- "帮我看看引言" / "我的研究问题对吗"
- "这个问题值不值得研究" / "我的contribution是什么"

**谜题发现类**
- "我发现了一个奇怪现象" / "实验结果出乎意料"
- "为什么会这样" / "这算不算一个好问题"

---

## 四种使用模式

### Mode A — 新选题引导
**场景**：有想法但没有清晰问题，从零开始找研究问题。

**AI行为**：
1. 让用户描述感兴趣的领域/现象
2. 引导识别这是议题/难题/疑问的哪种
3. 带用户走一遍三步提问法
4. 用沙漏四维度收口
5. 做聊天测试
6. 输出选题卡片

---

### Mode B — 论文叙事重构
**场景**：已有实验结果，但论文写成了"方法介绍"，缺乏研究问题驱动。

**AI行为**：
1. 让用户描述：做了什么实验、得到了什么结果、最关键的发现是什么
2. 挖掘被埋藏的谜题（通常藏在消融实验或对比实验里）
3. 把"我们提出了方法X，指标提升了Y%"重构为"为什么会出现现象Z？"
4. 重新提炼研究问题（带问号的疑问句）
5. 给出引言叙事结构建议（谜题→问题→本文贡献）
6. 输出叙事重构卡片

**诊断问题**（引导用户的核查问题）：
- 你的消融实验里，哪个因素贡献最大/最出乎意料？
- 你的基线对比里，最大的差距是什么？为什么会有这个差距？
- 如果有人问"你的工作解释了什么"，你能用一句话回答吗？
- 你的引言第一段，能让审稿人想问"为什么"吗？

---

### Mode C — 引言诊断
**场景**：已经写了引言，想知道哪里有问题。

**AI行为**：
1. 让用户粘贴或描述引言内容
2. 用三个标准逐一检查：
   - ✅ 是否有谜题（让人想知道答案的奇怪现象）
   - ✅ 是否有清晰的研究问题（疑问句，有问号）
   - ✅ 贡献是否回答了研究问题（而非只是"提出了方法X"）
3. 指出具体问题，给出修改方向
4. 可选：给出修改后的引言框架

**引言三要素检查**：
```
谜题：___（让读者想问"为什么"的现象）
研究问题：___（有问号的疑问句）
贡献：___（直接回答研究问题）
```

---

### Mode D — 谜题发现
**场景**：做实验发现了奇怪现象，不确定是否值得深挖。

**AI行为**：
1. 让用户描述这个"奇怪"的地方
2. 做谜题测试：如果说出来，别人第一反应是"为什么？"还是"哦"？
3. 判断谜题质量：
   - 是否违反常识/直觉？
   - 是否与已有理论不符？
   - 是否有多个可能的解释？
4. 如果是好谜题：帮助用转化为研究问题
5. 如果不够强：建议怎么找到更好的谜题角度

## 核心方法论（来源：《写作是门手艺》第7章，刘军强）

### 三类问题，只有一类能写论文

| 类型 | 英文 | 特点 | 能否直接写论文 |
|------|------|------|---------------|
| 议题 | Issue / Topic | 宽泛领域，无问号，范围庞大 | ❌ 不能，"老虎吃天，无从下嘴" |
| 难题 | Problem | 现实困境，无问号，需要行动干预 | ⚠️ 勉强接受，但不高分 |
| 疑问 | Question | 聚焦、有问号、令人好奇 | ✅ 合格研究问题 |

**核心判断**：你的选题里有没有真正的"？"，不是陈述句。

---

### 三步提问法

```
第一步：我要研究（一个宽泛的议题）

第二步：具体而言，我想聚焦于以下疑问：
  ① 为什么有的______，有的______却______？
     （比较差异，发现反常，产生问号）
  ② 什么因素影响了这一结果？
     （假设原因，寻找自变量）
  ③ 这些因素和结果之间的作用机制是怎样的？
     （解释路径，讲清因果链）

第三步：回答上述疑问，有助于帮助______解决______
  （价值关联：谁受益？解决什么问题？）
```

**第二步是核心，最难，也最值钱。**

---

### 沙漏写作法（从大到小，再从小到大）

```
[宽泛领域/议题]
      ↓ 聚焦（从大到小）
[具体研究问题]
      ↓ 扩展（从小到大）
[更大意义/贡献]
```

**四维收口**（把大问题压缩到可驾驭）：

| 维度 | 操作 | 目的 |
|------|------|------|
| Who（研究对象） | 缩小群体/模型/样本范围 | 可控性 |
| When（时间范围） | 限定时间段 | 避免淹没在历史数据中 |
| Where（场景/地域） | 限定地理区域或应用场景 | 可比性 |
| What（问题切面） | 聚焦某个具体方面 | 深度 |

**收口标准**：收缩到你能驾驭（manageable）的程度为止。
**核心价值观**：**片面而深刻 > 全面而肤浅**。

---

### 谜题测试（判断问题是否有趣）

> 你说出这个问题，听众的第一反应是"**为什么？**"（睁大眼睛）还是"**哦**"（漠然）？

- 好谜题：违反常识、与理论不符、有多个竞争答案
- 坏问题：平淡无奇、答案显而易见、无法引发讨论

---

### 聊天测试

> 你跟别人吃饭时，提出这个问题，大家是热烈讨论，还是蜻蜓点水忽略过去？

前者：好问题。后者：回去重选或换角度。

## AI 在不同模式下的输出格式

### Mode A 输出：选题卡片
```
领域 → 子领域 → 研究问题（完整路径）
三步提问法（填充结果）
沙漏收口（四维度）
聊天测试自评
```

### Mode B 输出：叙事重构卡片
```
原叙事（方法导向）→ 新叙事（问题导向）
找到的谜题
重构后的研究问题
引言结构建议：谜题→问题→本文贡献
```

### Mode C 输出：引言诊断报告
```
三要素检查结果（谜题/研究问题/贡献）
存在的问题（具体指出）
修改建议
可选：修改后引言框架
```

### Mode D 输出：谜题评估报告
```
谜题测试结果
谜题质量判断
转化为研究问题的方向
如需改进的建议
```

## 常见错误模式（跨领域通用）

| 错误 | 识别方式 | 修复方向 |
|------|----------|----------|
| 把领域当问题 | 选题是陈述句，没有"？" | 加问号，用三步提问法转化 |
| 把难题直接当研究问题 | 选题是"XXX问题研究" | 增加一个真实的问号 |
| 方法导向而非问题导向 | 论文重心在"我提出了方法X" | 找到被方法X回答的那个"为什么" |
| 问题太大无法驾驭 | 三万字写不完 | 用沙漏四维度收口 |
| 问题答案显而易见 | 谜题测试失败（反应是"哦"） | 找角度的反常处，发现悖论 |
| 贡献没有回答研究问题 | 引言问的和结论答的是两件事 | 对齐研究问题和贡献 |
